# 安全策略

## 支持范围

当前仅对最新公共版本和默认分支上的安全问题进行响应。

## 报告方式

如果你发现以下问题，请不要直接公开提交带利用细节的 Issue：

- 任意代码执行
- 本地文件越权读取
- 安装器篡改或提权
- 默认端口暴露导致的未授权访问
- 凭据、密钥或用户数据泄露

请通过私下渠道联系维护者，并至少提供：

- 影响版本
- 复现步骤
- 预期结果与实际结果
- 日志、截图或最小 PoC（概念验证）

## 处理原则

- 先修复，再公开披露
- 如需发布补丁版本，遵循 `SemVer`（语义化版本）并在 `CHANGELOG.md` 中记录
- 安全修复应优先保证升级可覆盖旧版本，并尽量保留用户数据
